# Contributing

Contributions are welcome through GitHub issues and pull requests.

- Do not submit customer documents, company templates, internal paths, API keys, translated business output, or other restricted data.
- Tests and screenshots must use synthetic and de-identified fixtures.
- Keep the service loopback-only unless an authenticated deployment design is reviewed separately.
- Preserve the two workflow modes and the existing project/result contracts.
- Run the public security scanner and relevant tests before opening a pull request.

By submitting a contribution, you represent that you have the right to provide it under the repository's `AGPL-3.0-only` license.
